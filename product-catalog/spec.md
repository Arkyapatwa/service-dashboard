# product-catalog

## Purpose
The product-catalog service manages the inventory of items available for purchase in the store. It provides fast, highly available read access to item details, categories, pricing, and real-time inventory counts for frontend browsing and checkout validation.

## Responsibilities
- Maintain product details, descriptions, images, and category hierarchies
- Query individual product details and browse paginated catalog lists
- Track and update real-time stock levels for stockkeeping units (SKUs)
- Reserve inventory temporarily during the checkout pipeline execution
- Release or commit inventory reservations based on order finalization

## API Endpoints

### GET /products
- **Purpose:** Returns a paginated list of active products with optional category filtering.
- **Request:** Query parameters: page (integer), limit (integer), category (string).
- **Response:** JSON body with matching products array and pagination metadata.
- **Auth:** none

### GET /products/{product_id}
- **Purpose:** Retrieves comprehensive details for a single product identifier.
- **Request:** Path parameter product_id string.
- **Response:** JSON body with complete product data and categories.
- **Auth:** none

### POST /products/reserve
- **Purpose:** Atomically decrements available stock and places a temporary reservation on items.
- **Request:** JSON body with reservation_id and items array.
- **Response:** JSON body with status, reservation_id, and expiration timestamp.
- **Auth:** required (Service-to-service authorization token required)

### POST /products/release
- **Purpose:** Releases a temporary inventory reservation, restoring the stock count.
- **Request:** JSON body with reservation_id.
- **Response:** JSON body confirming released status.
- **Auth:** required (Service-to-service authorization token required)

## Data Model
- **Product**
  - id: string — Unique identifier for the product (prefixed with prd_)
  - name: string — Display name of the item
  - description: string — Comprehensive written text describing the product
  - price: numeric — Fixed monetary value in USD
  - sku: string — Unique stock keeping unit code
  - stock_count: integer — Physical units available in the warehouse
  - categories: list[string] — Associated text tags for navigation sorting
- **InventoryReservation**
  - id: string — Unique identifier for the reservation lock (prefixed with res_)
  - product_id: string — Reference identifier matching the reserved product
  - quantity: integer — Amount of stock isolated for this specific transaction
  - expires_at: datetime — Deadline timestamp after which the reservation self-voids

## Dependencies
- External systems: MongoDB (document store optimized for catalog reads), Redis (distributed locks for reliable inventory atomicity)

## Non-functional Requirements
- Expected throughput: 500 catalog read queries per second; 50 reservation requests per second
- Latency targets: Catalog browsing endpoints under 50ms (P95); inventory reservation calls under 30ms (P99)
- Auth / authz rules: Public endpoints do not validate authentication; mutations and reservations require signed inter-service Bearer tokens generated by the infrastructure system
